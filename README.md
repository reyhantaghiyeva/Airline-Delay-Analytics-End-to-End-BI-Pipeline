# ✈️ Airline Delay Analytics — End-to-End BI Project

##  Project Overview
This project is an end-to-end Business Intelligence solution that analyzes airline delay patterns using real-world data.

It demonstrates a complete data pipeline from raw dataset processing to actionable business insights through visualization.

---

## Objective
To analyze airline delay causes and performance metrics in order to identify:
- Delay patterns by airline and airport
- Seasonal trends in flight delays
- Key factors affecting delays (weather, carrier, NAS, etc.)

---

##  Tech Stack
- **Python** (Pandas, NumPy)
- **SQL Server**
- **SSMS**
- **Power BI**
- **Kaggle Dataset**

---

##  Data Engineering Process

### 1. Data Cleaning (Python)
- Removed null values and duplicates
- Standardized column formats
- Handled missing data

### 2. ETL Process
- Extracted raw data using Pandas
- Transformed data into fact and dimension tables
- Loaded data into SQL Server

### 3. Data Modeling (Star Schema)
- Fact Table: Flight delays and performance metrics
- Dimension Tables: Airline and Airport information

---

## 📊 SQL Analysis
Performed analytical queries such as:
- Total flights per airline
- Delay ratio per carrier
- Monthly delay trends
- Top performing/underperforming airports

---

## 📈 Power BI Dashboard
Created interactive dashboard including:
- KPI cards (total flights, delay rate)
- Airline performance comparison
- Monthly trend analysis
- Airport-level insights

---

## 🔍 Key Insights
- Certain airlines consistently show higher delay ratios
- Weather and late aircraft are major delay factors
- Seasonal patterns strongly affect flight performance
